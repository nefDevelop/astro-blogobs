## Diferencia entre Markdown y MDX
- Markdown (MD) es un lenguaje de marcado ligero que permite a los usuarios escribir documentos en un formato de texto plano y luego convertirlos a HTML formateado. Es muy popular por su sintaxis simple y fácil de usar, siendo especialmente adecuado para escribir documentación y artículos de blog.
- MDX es un formato que extiende la sintaxis de Markdown, permitiendo insertar código JSX de manera fluida dentro de documentos Markdown. Con MDX, los usuarios pueden incrustar componentes React en sus documentos, logrando así una mayor interactividad y dinamismo.

| Característica | Markdown | MDX |
| :--- | :--- | :--- |
| Sintaxis básica | Soportada (CommonMark) | Soportada (CommonMark) |
| Etiquetas HTML | Soportadas | Soportadas (como JSX) |
| Importación de componentes | No soportada | Soportada (import) |
| Datos dinámicos | No soportada | Soportada (expresiones JS) |
| Personalización de estilos | Limitada (class/style) | Flexible (className/CSS-in-JS) |

## Uso de JSX

También puedes escribir HTML/JSX directamente:
```

  Este es un bloque div con estilo personalizado, utilizando clases de Tailwind CSS.
</div>
```
